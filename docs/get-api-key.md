# 국세청 API 서비스키 발급 가이드

본 서버를 **라이브 모드**로 사용하려면 공공데이터포털 국세청 API 서비스키가 필요합니다.
(`DEMO_MODE`면 키 없이도 가상 사업자번호로 도구를 체험할 수 있습니다.)

## 1. 공공데이터포털 가입
- https://www.data.go.kr 에서 회원가입

## 2. ⚠️ API 활용신청 — 두 서비스를 **각각 별도로** 신청해야 합니다

> **함정(실증 2026-07-08):** 한 서비스키로 "진위확인"과 "상태조회"가 **동시에 열리지 않습니다.** 두 서비스를 각각 활용신청해야 둘 다 사용 가능합니다.

국세청 사업자 API는 공공데이터포털에서 다음 형태로 제공됩니다:
- **사업자등록정보 진위확인** — `/validate` 엔드포인트
- **사업자등록정보 상태조회** — `/status` 엔드포인트

각각의 서비스 페이지에서 **[활용신청]** 버튼을 눌러 신청하세요.
- 서비스 페이지: https://www.data.go.kr/data/15081808/openapi.do
- 활용 목적 등 필수 항목 입력

## 3. 승인 대기
- 일반적으로 **1~2시간 이내** 승인. 여유 있게 잡아도 24시간.
- `마이페이지 → Open API → 활용신청 현황`에서 상태 확인.

## 4. 서비스키 확인
승인되면 같은 메뉴에서 인증키 발급:
- **Encoding 키**(URL 인코딩형) / **Decoding(일반) 키**(코드에서 통상 사용)
- 본 서버는 Decoding 키를 권장합니다(hex 64자리).

## 5. `.env` 설정
프로젝트 루트에 `.env` 파일을 만들고(이미 `.gitignore`에 추적 제외됨):
```
NTS_SERVICE_KEY=여기에_Decoding_키_입력
```

> **주의:** 서비스키는 절대 커밋·로그·코드에 노출하지 마세요. `.env`로만 관리합니다.
> 본 서버는 serviceKey를 자동으로 URL 쿼리로 전송하므로 키 값 그대로 넣으면 됩니다.

## 검증
```bash
node dist/index.js   # DEMO_MODE=1 없이, .env 있으면 라이브 모드
# 별도 에이전트/Inspector로 verify_business 호출 시 실제 국세청 응답 확인
```

진위/상태 중 하나만 신청한 상태라면, 해당 도구만 라이브 동작하고 나머지는 `SERVICE_NOT_ACTIVATED` 에러를 반환합니다 — 이 경우 누락된 서비스를 추가 신청하세요.
