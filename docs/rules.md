# 판정 규칙표 — `check_invoice_eligibility`

> **면책**: 본 판정은 국세청 공공데이터 기준의 **참고 판정**이며, 세무 자문이 아닙니다. 법적 효력이 있는 증명은 홈택스 발급 문서를 참조하시고, 실무 판단은 세무사 확인을 권장합니다.

## 입력 (국세청 상태조회 응답)
- `b_stt` / `b_stt_cd`: 사업 상태. **01**=계속사업자 / **02**=휴업자 / **03**=폐업자
- `tax_type` / `tax_type_cd`: 과세유형

## 과세유형 분류 (보수적 키워드 매칭 — 코드 의존도 최소)
`tax_type` 문자열에서 키워드 포함 여부로 분류. 순서 중요(면세→간이→비과세→일반).

| 분류 | 키워드 | `tax_type_cd` fallback |
|---|---|---|
| exempt(면세) | `면세` | `03` |
| simplified(간의) | `간이` | `02` |
| nontaxable(비과세) | `비과세` | `04` |
| general(일반) | `일반` (또는 위 해당 없음) | `01` |

> `tax_type_cd`는 **2자리 문자열**(`"01"` 등, 실증 2026-07-08). 공공데이터 문서 기반이나 변형(성실신고 등)이 있을 수 있어 **문자열 키워드를 1차**, 코드를 fallback으로 사용한다.

## 판정표 (상태 × 과세유형 → 판정)
| 상태 | 과세유형 | 판정코드 | basis(근거) |
|---|---|---|---|
| 03 폐업 | * | `UNAVAILABLE` | 폐업자는 세금계산서 발행이 불가능 |
| 02 휴업 | * | `SUSPENDED` | 휴업 중, 거래 제한·확인 필요 |
| 01 계속 | general | `TAX_INVOICE_OK` | 일반과세자, 세금계산서 수취 가능 |
| 01 계속 | simplified | `TAX_INVOICE_OK` | 간이과세자(2023년 세금계산서 의무화), 수취 가능 |
| 01 계속 | exempt | `CASH_INVOICE_EXEMPT` | 면세사업자, 부가세 과세 대상 아니라 세금계산서 대신 계산서 |
| 01 계속 | nontaxable | `UNCLEAR` | 비과세사업자, 품목·거래에 따라 확인 필요 |
| 01 계속 | unknown | `UNCLEAR` | 과세유형 미확인 |

## 결과 코드 + 라벨
| 코드 | 한국어 라벨 |
|---|---|
| `TAX_INVOICE_OK` | 세금계산서 수취 가능 |
| `CASH_INVOICE_EXEMPT` | 계산서 대상(면세사업자) |
| `UNAVAILABLE` | 발행 불가(폐업) |
| `SUSPENDED` | 휴업 중 |
| `UNCLEAR` | 확인 필요 |

## 출력 (도메인 원칙: 반드시 basis 동봉)
```
{ eligibility: <코드>, label: <라벨>, basis: <근거>, disclaimer: <면책 문구> }
```

## 규칙 우선순위
1. 폐업(03) → 무조건 `UNAVAILABLE` (과세유형 무관)
2. 휴업(02) → `SUSPENDED`
3. 계속(01) → 과세유형 따라 판정
